# GEC6818 无人车库管理系统 - 技术报告

## 一、项目概述

### 1.1 项目背景

随着城市化进程加快，停车难问题日益突出。传统的人工管理车库存在效率低、人力成本高、易出错等问题。本项目基于粤嵌 GEC6818 开发板，设计并实现了一套无人车库管理系统，实现了车辆进出的自动化管理。

### 1.2 项目目标

- 实现车辆入库、出库的自动化管理
- 支持摄像头车牌识别（ALPR）
- 支持 RFID 刷卡自动出入库
- 提供图形化用户界面
- 实现停车费用自动计算
- 支持数据记录和查询

### 1.3 开发环境

| 项目 | 说明 |
|------|------|
| 开发板 | 粤嵌 GEC6818（ARM Cortex-A53） |
| 操作系统 | Linux（嵌入式） |
| 开发语言 | C++17 |
| UI 框架 | Qt 5.7 Embedded |
| 图像处理 | OpenCV 3.4 |
| 车牌识别 | HyperLPR |
| 构建工具 | CMake 3.10 |
| 交叉编译器 | arm-linux-g++ 5.4.0 |

## 三、功能模块详解

### 3.1 图形解锁模块

#### 3.1.1 功能描述

锁屏界面提供 3x3 网格图案解锁功能，用户通过触摸拖动绘制解锁图案，至少连接 4 个点才能解锁。

#### 3.1.2 技术实现

**核心类：** `PatternLock`

**关键算法：**
- 网格布局：根据屏幕尺寸动态计算 9 个圆点位置，形成正方形网格
- 碰撞检测：计算触摸点与圆点的距离，判断是否选中
- 图案验证：记录选中点的顺序，与预设密码比对

**默认密码：** 1→2→3→6→9（索引 0→1→2→5→8）

**代码实现：**
```cpp
// 碰撞检测
int PatternLock::hitTest(const QPoint &pos) const
{
    int hitRadius = dotRadius_ + 15;
    for (int i = 0; i < points_.size(); ++i) {
        double dx = points_.at(i).x() - pos.x();
        double dy = points_.at(i).y() - pos.y();
        if (std::sqrt(dx * dx + dy * dy) <= hitRadius) {
            return i;
        }
    }
    return -1;
}
```

#### 3.1.3 界面设计

- 背景色：白色（#f5f5f5）
- 圆点颜色：灰色（未选中）、蓝色（选中）
- 连线颜色：蓝色（绘制中）、绿色（成功）、红色（失败）
- 圆点大小：根据屏幕尺寸动态计算

void CameraViewer::startCamera(const QString &dev, int width, int height)

### 3.3 车牌识别模块（ALPR）

#### 3.3.1 功能描述

自动识别车辆车牌号，用于车辆身份确认。

#### 3.3.2 技术实现

**技术栈：** HyperLPR（深度学习车牌识别）

**工作流程：**
```
摄像头抓拍 → 保存照片 → 启动 ALPR 子进程 → 命名管道通信 → 读取识别结果
```

**通信方式：** 命名管道（FIFO）

```cpp
// 创建命名管道
mkfifo(PIPE_INBOUND, 0666);

// 启动 ALPR 子进程
alprProcess_->start(alprBin, QStringList() << photoPath << pipePath);

// 从管道读取结果
fd_set readfds;
struct timeval tv;
tv.tv_sec = 15;
select(pipeReadFd_ + 1, &readfds, NULL, NULL, &tv);
read(pipeReadFd_, buf, sizeof(buf));
```

**识别流程：**
1. 用户点击"车辆入库"或"车辆出库"
2. 摄像头抓拍照片保存到 `/tmp/vehicle_snap.jpg`
3. 启动 ALPR 子进程进行识别
4. 通过命名管道 `/tmp/alpr_inbound` 或 `/tmp/alpr_outbound` 传递结果
5. 主程序读取识别结果

### 3.4 RFID 自动出入库模块

#### 3.4.1 功能描述

通过 RFID 读卡器自动识别车辆，实现无人值守的自动出入库。

#### 3.4.2 硬件配置

| 项目 | 说明 |
|------|------|
| RFID 模块 | HW-033 |
| 串口设备 | `/dev/ttySAC1`（UART1） |
| 波特率 | 9600 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |

#### 3.4.3 协议格式

**请求帧：**
```
07 02 41 01 52 BCC 03
```

**防碰撞帧：**
```
08 02 42 02 93 00 BCC 03
```

**应答帧：**
```
XX XX 00 [4字节卡号] [校验]
```

#### 3.4.4 技术实现

**核心类：** `RfidReader`（继承 `QThread`）

```cpp
class RfidReader : public QThread
{
    Q_OBJECT
signals:
    void cardDetected(const QString &cardId);
protected:
    void run() override;
};
```

**工作流程：**
1. 独立线程持续读取串口数据
2. 发送请求命令检测是否有卡
3. 发送防碰撞命令获取卡号
4. 发射 `cardDetected` 信号
5. 主界面接收信号，查询卡号-车牌映射
6. 自动判断入库/出库并执行相应操作

**卡号-车牌映射：**
```cpp
cardPlateMap_["83533443"] = "贵B91VIP";
```

### 3.5 数据存储模块

#### 3.5.1 功能描述

记录车辆进出信息，支持查询和统计。

#### 3.5.2 存储格式

**数据库文件：** `/frog/gate.db`（CSV 格式）

**数据格式：**
```
[车牌标识],[操作类型],[时间戳]
```

**示例：**
```
U+8D35B91VIP,inbound,2026-06-12 10:30:00
U+8D35B91VIP,outbound,2026-06-12 15:45:00
```

**车牌编码：**
- 中文字符转换为 Unicode 码点
- 示例："贵B91VIP" → "U+8D35B91VIP"

#### 3.5.3 业务逻辑

**入库校验：**
- 查询车牌最后状态
- 如果已在库内（inbound），拒绝重复入库
- 如果不在库内或已出库（outbound），允许入库

**出库校验：**
- 查询车牌最后状态
- 如果不在库内，拒绝出库
- 如果在库内（inbound），允许出库并计算费用

**计费规则：**
- 每秒 1 元（演示用）
- 出库时自动计算停车时长和费用

## 八、问题与解决方案

### 8.1 音画不同步

**问题：** 开机动画中视频和音频分开播放，导致不同步。

**解决方案：** 使用 mplayer 同时播放视频和音频（-ao alsa），或通过延时控制同步。

### 8.2 界面显示偏移

**问题：** 在 800x480 屏幕上，界面元素显示不全。

**解决方案：** 调整布局边距和间距，优化字体大小和按钮 padding。

### 8.3 RFID 卡号解析错误

**问题：** 车牌中的字母被错误解析为 Unicode 码点的一部分。

**解决方案：** 限定 Unicode 码点固定为 4 位十六进制数。

### 8.4 开机启动卡顿

**问题：** 在 `/etc/profile` 中配置开机启动，视频播放卡顿。

**解决方案：** 改为手动启动，或使用 `sleep` 延时等待系统初始化完成。
